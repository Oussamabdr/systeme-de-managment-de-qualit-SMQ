## 🚀 Deployment to Render.com

### **Option 1: Automatic Blueprint Deployment (Recommended)**

If your GitHub repository is connected to Render with Blueprint support:

1. **Render detects `render.yaml` automatically**
2. **Services deploy:**
   - ✅ Backend API (Node.js web service)
   - ✅ Frontend (Static site)

**Timeline:** 4-6 minutes total

- Build backend: 1-2 min
- Build frontend: 1-2 min
- Deploy + migrations: 1-2 min

### **Option 2: Manual Setup (If Blueprint not enabled)**

#### **Step 1: Deploy Backend API**

1. Go to [render.com](https://render.com)
2. **New → Web Service**
3. **Connect GitHub** and select your repository
4. **Settings:**
   - **Name:** `iso-qms-api`
   - **Environment:** `Node`
   - **Build Command:** `npm install && npm run db:generate`
   - **Start Command:** `npm start`
   - **Root Directory:** `backend`
5. **Environment Variables:**
   - `NODE_VERSION`: `20`
   - `DATABASE_URL`: [Set from Render PostgreSQL or external DB]
   - `JWT_SECRET`: [Auto-generated by Render]
   - `JWT_EXPIRES_IN`: `1d`
   - `CORS_ORIGIN`: Will be set after frontend deploys
6. **Post-Deploy Command:** `npx prisma migrate deploy && npm run db:seed`
7. **Deploy** → Wait for success (2-3 min)

#### **Step 2: Deploy Frontend**

1. **New → Static Site**
2. **Connect GitHub** and select your repository
3. **Settings:**
   - **Name:** `iso-qms-frontend`
   - **Build Command:** `npm install && npm run build`
   - **Publish Directory:** `frontend/dist`
   - **Root Directory:** `frontend`
4. **Environment Variables:**
   - `VITE_API_URL`: Will be set from backend URL
5. **Deploy** → Wait for success (1-2 min)

#### **Step 3: Link Services**

1. Go to **Backend API settings**
2. Update `CORS_ORIGIN` environment variable: `https://iso-qms-frontend-xxxx.render.com`
3. **Redeploy** backend

## 🛠️ Quick Troubleshooting

### **"CORS blocked" Error**

- **Fix:** Update Backend `CORS_ORIGIN` to match Frontend URL
- **Command:** Redeploy backend after updating env var

### **Frontend shows blank page**

- **Fix:** Check browser console (F12)
- **Common:** `VITE_API_URL` not set correctly
- **Solution:** Verify environment variable in Render dashboard

### **API calls timeout**

- **Fix:** Verify Backend is running: `https://iso-qms-api-xxxx.render.com/`
- **Check:** Backend logs in Render dashboard

### **Database migration failed**

- **Fix:** Ensure `DATABASE_URL` is valid PostgreSQL connection
- **Command:** Run `npx prisma migrate deploy` locally to debug

### **Language switching not working**

- **Fix:** Check localStorage in DevTools
- **Look for:** `qms_lang` key with value `fr` or `en`
- **Backend:** Restart browser cache (Ctrl+Shift+Delete)
